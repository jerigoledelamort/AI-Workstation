## 1. Категоризация проблем

### Категория A — обязательно исправить

Проблемы, которые блокируют работу, влияют на безопасность,
воспроизводимость или могут привести к потере данных.

---

#### A1. C1 — Docker демон не запущен

| Параметр | Оценка |
|----------|--------|
| **Категория** | A — блокирует работу |
| **Почему** | Docker нужен для Qdrant, Open WebUI, DevContainers. Без запущенного демона эти компоненты не работают. |
| **Риск если оставить** | Невозможность запустить контейнерные сервисы. Весь Phase 6 Roadmap заблокирован. |
| **Стоимость исправления** | Низкая. Запустить Docker Desktop + установить StartType=Automatic. |
| **Побочные эффекты** | Потребление ~4-8 GB RAM при работе Docker. |

---

#### A2. C2 — Ollama не зарегистрирован как служба

| Параметр | Оценка |
|----------|--------|
| **Категория** | A — надёжность, потеря доступности |
| **Почему** | Ollama работает как пользовательский процесс. При выходе из системы или перезагрузке — остановится. Нет автозапуска. Воспроизводимость состояния станции нарушена. |
| **Риск если оставить** | После перезагрузки AI-станция неработоспособна. Ручной запуск каждый раз. |
| **Стоимость исправления** | Низкая. Создать запланированную задачу или зарегистрировать службу. |
| **Побочные эффекты** | Ollama будет запускаться до входа пользователя. Минимальные. |

---

#### A3. H2 — OLLAMA_HOST не установлен

| Параметр | Оценка |
|----------|--------|
| **Категория** | A — безопасность |
| **Почему** | Без явного OLLAMA_HOST=127.0.0.1:11434 Ollama может привязаться к 0.0.0.0 при обновлении или смене конфигурации. Это exposing AI API в сеть. |
| **Риск если оставить** | Случайное открытие Ollama API в локальную сеть. Любое устройство в сети получит доступ к моделям без auth. |
| **Стоимость исправления** | Низкая. Установка переменной окружения. |
| **Побочные эффекты** | Нет. |

---

#### A4. H3 — Firewall DefaultInboundAction = NotConfigured

| Параметр | Оценка |
|----------|--------|
| **Категория** | A — безопасность |
| **Почему** | Неявная конфигурация firewall. NotConfigured означает default (Block), но не зафиксирована. Системное обновление или установка ПО может изменить default. |
| **Риск если оставить** | Теоретический — изменение default на Allow откроет все порты. |
| **Стоимость исправления** | Низкая. Set-NetFirewallProfile -DefaultInboundAction Block. |
| **Побочные эффекты** | Нет. Default уже Block — явная фиксация не меняет поведения. |

---

#### A5. H4 — Нет явных firewall правил для AI-портов

| Параметр | Оценка |
|----------|--------|
| **Категория** | A — безопасность (вместе с A4) |
| **Почему** | Порты 11434, 4000, 6333, 6334, 8080, 8000 — без явных inbound block-правил. Многоуровневая защита (Architecture.md, раздел 3) требует явных правил. |
| **Риск если оставить** | Если default изменится — все AI-сервисы доступны извне без auth. |
| **Стоимость исправления** | Низкая. 6 команд New-NetFirewallRule. |
| **Побочные эффекты** | Нет. Правила только block inbound. |

---

#### A6. H5 — Две версии Python (3.10 + 3.14)

| Параметр | Оценка |
|----------|--------|
| **Категория** | A — воспроизводимость |
| **Почему** | Две версии Python создают неоднозначность. `where.exe python` возвращает 3 записи. Скрипты могут запускаться с разными версиями. Воспроизводимость нарушена. |
| **Риск если оставить** | Случайный запуск скрипта под Python 3.14 вместо 3.10. Несовместимость пакетов. Тихие ошибки. |
| **Стоимость исправления** | Средняя. Удалить Python 3.14 или явно зафиксировать версию через uv. Требует решения пользователя. |
| **Побочные эффекты** | Удаление 3.14 может затронуть ПО, зависящее от неё. |

#### A7. H7 — Git core.autocrlf не настроен

| Параметр | Оценка |
|----------|--------|
| **Категория** | A — воспроизводимость |
| **Почему** | Без core.autocrlf Git использует default (false на Windows после 2.x). В DevContainers (Linux) и на Windows будут разные line endings. Конфликты в коммитах. |
| **Риск если оставить** | CRLF/LF конфликты в DevContainers. Diff-шум. Возможные ошибки выполнения скриптов. |
| **Стоимость исправления** | Низкая. `git config --global core.autocrlf true` (Windows) или `.gitattributes` в проекте. |
| **Побочные эффекты** | Минимальные. |

#### A8. M2 — LiteLLM установлен в global pip

| Параметр | Оценка |
|----------|--------|

| **Побочные эффекты** | Нужно обновить скрипты запуска для использования venv. |

## 3. Приоритет исправлений

Приоритет определяется: Criticality > Security > Reproducibility > Convenience.

| Приоритет | Проблема | Категория | Обоснование приоритета |
|-----------|----------|-----------|------------------------|
| **P0** | C1 — Docker демон | A | Блокирует контейнерные сервисы |
| **P0** | C2 — Ollama служба | A | Блокирует автозапуск inference |
| **P1** | H2 — OLLAMA_HOST | A | Безопасность: явная привязка localhost |
| **P1** | H3 — Firewall default | A | Безопасность: явная конфигурация |
| **P1** | H4 — Firewall AI-порты | A | Безопасность: block inbound |
